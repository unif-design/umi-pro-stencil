umi-pro-stencil · 目录结构标准

@umijs/max + Ant Design Pro v6 · Web 端代码归位，及组件 / 页面内部结构

核心原则：私有就近，复用上提。 只在某组件 / 页面内用的方法、工具、常量、类型、子组件 → 放它自己目录里（私有文件用单数、子组件直接平铺）； 一旦 ≥2 处复用，再提到 src/ 公共目录（services / hooks / utils / constants / types / components，复数）。

SRC 顶层 src/ 与根级目录

每类代码唯一归属；公共能力集中在这些大文件夹。

src/
├─ pages/路由页面，一页一目录（见"页面内部"）
├─ components/跨页面复用的业务组件（antd/Pro 封装）
├─ services/接口：<模块>/XxxController.ts + typings.d.ts + index.ts
├─ hooks/跨组件复用的 hook（useXxx.ts）
├─ models/全局 / 跨组件状态（umi model + useModel）
├─ types/全局 TS 类型（*.d.ts）
├─ utils/ · constants/ · assets/纯函数 · 常量枚举 · 静态资源
└─ access.ts · app.tsx权限 · 运行时（getInitialState/layout/request）
config/构建/路由/插件：config.ts + routes.ts + defaultSettings.ts + proxy.ts
mock/本地接口 mock（遵循 {code,data,message} 信封）
src/.umi*/生成物，勿动（gitignore）

组件内部 components/<Name>/

复用业务组件；私有内容就近，子组件平铺，外部只从 index 引。

ArticleCard/
├─ index.tsx唯一对外入口
├─ ArticleCard.tsx主组件（与目录同名）
├─ ArticleTag.tsx私有子组件（直接平铺，不套 components/）
├─ hook.ts私有方法 / 逻辑（单数）
├─ util.ts私有工具
├─ constant.ts私有常量
├─ type.ts私有类型（含 Props）
└─ index.style.ts样式（antd-style createStyles）

极简组件可只留 index.tsx。外部 import ArticleCard from '@/components/ArticleCard'，不深引内部文件。

页面内部 pages/<Page>/

路由页面；与组件同构——私有内容就近，子组件平铺。

Article/
├─ index.tsx页面入口（config/routes.ts 注册）
├─ ArticleModal.tsx页面私有子组件（平铺）
├─ hook.ts页面私有方法 / 逻辑
├─ util.ts页面私有工具
├─ constant.ts页面私有常量
├─ type.ts页面私有类型
└─ index.style.ts页面私有样式

接口走 src/services/，页面只 import。某块复用 ≥2 次才提到 src/。

目录 文件 跨范围复用 → src/ 公共目录（复数） 私有 → 就近、单数、平铺 配置 生成物
